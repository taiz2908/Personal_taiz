-> Project Mgmt & Comprehensive tool written in Java 
-> pom.xml - Project Object Model , Extensible Markup Language
-> In project we have one "parent pom" , and every module have separate pom.

Old - ANT
New - Gradel
Dependencies- Its refers to the Java liberares that are needed for the project.

Main file is "pom.xml" contains info about the project & various configuration details.
Also it contains 
- project Info
- metadata
- goals 
- plugins
- Dependencies etc.

Ex:
    <groupId>sathya</groupId>
    <artifactId>TestProject</artifactId>
    <version>1.5</version>
    <packaging>war</packaging>
    <name>myproj</name>

Maven Goal -
 mvn clean: to clean the repo / Previous project to be clean
 mvn site - Aslo it will generate reports.
 mvn compile: to compile the source code of the project , Executable files.
 mvn Test: to apply test cases. 
 mvn package: to generate the package,executable files, i.e .war / .jar
 mvn install: install the package in the local repo in mvn i.e. ".m2"
 mvn deploy: to deplot in nexus repo
 
Whatever artifact generated it'll store under "target" folder.
In Pom.xml: 
<distributionManagement> 
section we need to define where we should push our artifact.
<id> should match with setting.xml within .m2 folder.
<snapshotRepository> related to DEV env.
